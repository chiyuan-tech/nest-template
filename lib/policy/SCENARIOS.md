# Policy 系统所有场景表格

## ⚠️ 重要：等级 0 + 免费次数 > 0 + 积分足够的情况

当用户满足以下条件时：
- ✅ 等级 `≤ 0`
- ✅ 免费次数 `> 0`
- ✅ 积分足够

**会根据表单参数分为两种情况：**

| 表单参数 | 结果 | 消耗资源 | 说明 |
|---------|------|---------|------|
| **满足限制**<br/>(480p + 5s) | ✅ 使用免费券 | 消耗 1 次免费次数 | **优先使用免费券**，即使积分足够也不会用积分 |
| **不满足限制**<br/>(720p 或 10s) | ✅ 使用积分 | 消耗积分 | 免费券无法使用，使用积分生成，**免费次数保留** |

**关键点：**
1. 🎁 **满足限制时，优先使用免费券**（即使积分足够）
2. 💰 **不满足限制时，使用积分**（免费次数保留）
3. 🔄 **用户无法选择**使用免费券还是积分，系统自动决定

## 详细场景表格

### 场景 1：未登录用户

| 条件 | 值 | 说明 |
|------|-----|------|
| **isSignedIn** | `false` | 用户未登录 |
| **level** | - | 不检查 |
| **free_times** | - | 不检查 |
| **表单参数** | 任意 | 不检查 |
| **积分** | - | 不检查 |

**决策结果：**
- ✅ `allowed`: `false`
- 🔘 `primaryAction`: `'signin'`
- 📝 `buttonText`: `'GENERATE'`
- 🏷️ `costBadge`: `0`
- 📌 `denyReason`: `'not_signed_in'`
- 💬 `message`: `undefined`

**用户操作：** 点击按钮 → 弹出登录弹窗

---

### 场景 2：有优惠券资格 + 满足限制条件

| 条件 | 值 | 说明 |
|------|-----|------|
| **isSignedIn** | `true` | 用户已登录 |
| **level** | `≤ 0` | 免费用户 |
| **free_times** | `> 0` | 有免费次数 |
| **表单参数** | 满足限制 | 例如：480p + 5s |
| **积分** | 任意（不检查） | **即使积分足够，也优先使用免费券** |

**决策结果：**
- ✅ `allowed`: `true`
- 🔘 `primaryAction`: `'generate'`
- 📝 `buttonText`: `'GENERATE'`
- 🏷️ `costBadge`: `'free'`
- 📌 `voucherEligible`: `true`
- 📌 `voucherRestricted`: `false`
- 💬 `message`: `undefined`

**用户操作：** 点击按钮 → 使用免费券生成视频

**重要说明：**
- ⚠️ **即使用户积分足够，系统也会优先使用免费券**
- ⚠️ **不会消耗积分，只消耗免费次数**

**示例：**
- 选择 480p + 5s，有 1000 积分 → ✅ 使用免费券生成（不消耗积分）
- 选择 480p + 5s，有 0 积分 → ✅ 使用免费券生成（不消耗积分）

---

### 场景 3：有优惠券资格 + 不满足限制 + 积分足够

| 条件 | 值 | 说明 |
|------|-----|------|
| **isSignedIn** | `true` | 用户已登录 |
| **level** | `≤ 0` | 免费用户 |
| **free_times** | `> 0` | 有免费次数（但参数不满足限制，无法使用） |
| **表单参数** | 不满足限制 | 例如：720p 或 10s |
| **积分** | `≥ 所需积分` | 积分足够 |

**决策结果：**
- ✅ `allowed`: `true`
- 🔘 `primaryAction`: `'generate'`
- 📝 `buttonText`: `'GENERATE'`
- 🏷️ `costBadge`: `所需积分数字` (如 `70`, `140`)
- 📌 `voucherEligible`: `true`
- 📌 `voucherRestricted`: `true` ⚠️
- 📌 `requiredCredits`: `所需积分数`
- 💬 `message`: `'Free vouchers only support 480p / 5s. Using credits instead.'`

**用户操作：** 点击按钮 → 使用积分生成视频

**重要说明：**
- ⚠️ **免费券无法使用**（因为参数不满足限制）
- ✅ **使用积分生成**（因为积分足够）
- 💡 **免费次数保留**（不消耗，因为使用了积分）

**示例：**
- 选择 720p + 5s，有 100 积分 → ✅ 使用 70 积分生成（免费次数保留）
- 选择 480p + 10s，有 100 积分 → ✅ 使用 60 积分生成（免费次数保留）

---

### 场景 4：有优惠券资格 + 不满足限制 + 积分不足

| 条件 | 值 | 说明 |

- 📌 `requiredCredits`: `所需积分数`

- 📌 `voucherEligible`: `false`
- 📌 `voucherRestricted`: `false`
- 📌 `requiredCredits`: `所需积分数`
- 💬 `message`: `undefined`

**用户操作：** 点击按钮 → 使用积分生成视频

**示例：**
- 付费用户，选择 720p + 10s，有 200 积分 → ✅ 使用 140 积分生成

### 场景 7：无积分定价配置（向后兼容）

| 条件 | 值 | 说明 |
|------|-----|------|
| **isSignedIn** | `true` | 用户已登录 |
| **level** | 任意 | 不检查 |
| **free_times** | 任意 | 不检查 |
| **表单参数** | 任意 | 不检查 |
| **积分定价** | `undefined` | 产品在 `products.ts` 中未配置 `creditPricing` 函数 |

**决策结果：**
- ✅ `allowed`: `true`
- 🔘 `primaryAction`: `'generate'`
- 📝 `buttonText`: `'GENERATE'`
- 🏷️ `costBadge`: `0`
- 📌 `requiredCredits`: `0`
- 💬 `message`: `undefined`

**用户操作：** 点击按钮 → 直接生成（不检查积分）

**触发条件：**
当产品在 `lib/policy/products.ts` 中没有配置 `creditPricing` 函数时，会触发此场景。

**代码逻辑：**
```typescript
// 4. Not using voucher, check credits
if (policy?.creditPricing) {
  // 有配置 → 检查积分（场景5或场景6）
} else {
  // 无配置 → 场景7：默认允许生成（向后兼容）
  decision.allowed = true;
  decision.primaryAction = 'generate';
  decision.buttonText = 'GENERATE';
}
```

**使用场景：**
1. 🆕 **新添加的产品**：尚未配置积分定价规则
2. 🎁 **特殊产品**：某些产品可能免费或使用其他计费方式
3. 🔄 **向后兼容**：避免因配置缺失导致功能不可用

**当前状态：**
- ✅ `seedance-1-pro`：已配置积分定价（不会触发此场景）
- ✅ `seedance-1-5-pro`：已配置积分定价（不会触发此场景）
- ⚠️ 如果未来添加新产品但忘记配置，会触发此场景

**注意事项：**
- ⚠️ 此场景主要用于向后兼容和开发阶段
- ⚠️ 生产环境中的产品应该都配置积分定价
- ⚠️ 如果产品需要积分检查，必须在 `products.ts` 中配置 `creditPricing` 函数

## 优惠券限制条件

### Seedance 1.0 Pro & 1.5 Pro
- ✅ **允许的分辨率**: `['480p']`
- ✅ **允许的时长**: `[5]` (5秒)
- ✅ **宽高比**: 无限制
- ✅ **模式**: 无限制 (text-to-video / image-to-video)

### 优惠券资格判断
```typescript
voucherEligible = (level <= 0) && (free_times > 0)
```

```
开始
  ↓
是否登录？
  ├─ 否 → 场景1: 要求登录
  └─ 是 ↓
     是否优惠券资格？(level ≤ 0 && free_times > 0)
        ├─ 是 ↓
        │   是否满足优惠券限制？
        │     ├─ 是 → 场景2: 使用免费券
        │     └─ 否 ↓
        │         积分是否足够？
        │           ├─ 是 → 场景3: 使用积分
        │           └─ 否 → 场景4: 要求升级
        └─ 否 ↓
            是否有积分定价配置？
              ├─ 是 ↓
              │   积分是否足够？
              │     ├─ 是 → 场景5: 使用积分
              │     └─ 否 → 场景6: 购买积分
              └─ 否 → 场景7: 允许生成（向后兼容）
```
